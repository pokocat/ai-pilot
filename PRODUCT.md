# 军师 · 产品说明

本文承接 `AGENTS.md` 中不必每次塞给 coding agent 的产品背景。工程约束、接口、构建、TODO 与变更日志仍以 `AGENTS.md` 为准。

## 1. 产品定位

**军师**：面向创始人 / CEO 的 AI 商业军师。产品价值分两条主线：

- **出谋（智库）**：通用军师 + 多位顾问型智能体，围绕战略诊断、增长、融资、竞品、组织等问题产出结构化咨询成果。
- **出活（智能体工坊）**：创作型智能体把战略落成品牌资产，如 IP、宣传片、海报、短视频和文案，并逐步支持训练“只懂你”的专属助手。

核心体验：

- **本命色**：首登选择金、绿、红、蓝、紫、铁之一，全站主题随之变化，可在「我的」重选。
- **Agent Memory**：顾问从企业档案、对话、资料和反馈里持续学习，前台称为「专属理解」，后台可配置学习策略。
- **智能体权益 / 产出额度**：底层支持 `free`、`unlock`、`metered`；前台用「可用 / 已启用 / 专项能力 / 方案与产出额度」表达。智能体价格展示统一用钻石格式：一次启用 `💎xN`，按次产出 `💎xN/次`。
- **企业事务操作系统**：以项目为主线，串起会话、版本化报告、知识库和长期记忆。对话可 @ 引用项目/报告/知识，报告可版本化和 diff，对话可汇总成纪要并沉淀知识。
- **多租户**：每个账号独立租户，业务数据行级隔离。

## 2. 信息架构

5-tab 用户动线（2026-07 重构）：**对话是入口，执行是结果**——参谋室诊断 → 认可方案 → 执行承接 → 回填复盘 → 战局刷新。

| 模块 | 面向用户的价值 |
|---|---|
| 战局 | 当前最重要的判断：军师判断（真实军师档案）、待补线索、下一步动作、三势判断入口、今日献策 |
| 执行 | 承接认可的方案：执行案卷（项目主线）、第 0 号军令（补资料）、一日节奏（军令/创作/复盘）、AI 创作发布（创作型智能体出活） |
| 对话 | 军师参谋室（第一入口）：总军师统筹 + 专业军师线程协同，结论回流主线；引用上下文、生成结构化成果和纪要 |
| 智库 | 军师判断的底座：方法论、资料库（AI 分类）、数据源绑定、模块 + Skill 市场、版本化报告与方案库 |
| 我的 | 企业档案、项目工作台、方案库、数据源、模块、军师社群、方案与产出额度、本命色 |
| 运营后台 | 用户、消耗、审计、智能体、套餐、模型和内容配置 |

产品原则（承接军师产品需求文档）：模块不是推荐商品，而是军师方案的功能化承接；Skill 是军师调用的方法能力包；数据绑定由军师按问题引导，不要求一次全绑；页面分布式浏览，底层围绕同一个用户案卷。

## 3. 企业事务操作系统

一条主线 + 四块能力，长在既有会话、记忆、成果之上。

| 能力 | 后端落点 | 前端落点 |
|---|---|---|
| 项目 | `routes/projects.ts`，`Session/Memory/Deliverable.projectId` | `packages/work/projects`、`packages/work/project` |
| 知识库 + 语义记忆 | `services/{embedding,retrieval,knowledge}.ts`，`routes/knowledge.ts` | 项目详情知识段，@ 引用候选 |
| 版本化报告 | `services/reports.ts`，`routes/reports.ts`，`ReportDoc/ReportVersion` | `packages/work/report`，方案库 `vN` 徽标 |
| @ 引用 | `buildGenContext` + `resolveReferences` + `Message.refsJson` | 对话页引用选择器和 chip |
| 对话汇总 | `services/summarize.ts`，`POST /sessions/:id/summarize` | 对话页「生成纪要」 |

关键产品规则：

- 检索采用语义向量 + 关键词混合召回；演示环境用本地确定性嵌入，生产可切 pgvector。
- 报告按「报告名 slug」归一，同名再产出或编辑即新版本，同内容哈希不重复成版。
- mock provider 也会把引用、知识、项目体现在产出里，便于无真实模型演示。
- 项目、报告、知识全部按 `tenantId` 隔离；引用解析只取当前用户/租户可见资料。

## 4. 产品文案口径

- 主路径避免“赠送 / 付费解锁 / 充值 / 最受欢迎 / 灵活付费”等促销口吻。
- 用户侧统一用「可用」「已启用」「专项能力」「产出额度」「方案与额度」「常用配置」。
- 智能体价格只露出交互型金额：`💎xN` / `💎xN/次`。后台、代码、接口仍可使用 `billing/price/credits`。
- Agent Memory 用户侧称为「专属理解」，不要直接暴露后台术语。
- AI 返回的 Markdown 要渲染成文档感内容，不把 `###`、`**`、`-` 原样展示给用户。

## 5. 升级方向

- **pgvector**：路径已实现，待真库验证；开启 `PGVECTOR_ENABLED=true` 后走 ANN 下推和向量列双写。
- **真实嵌入模型**：后台或 env 配 `embeddingModel` + 真实 key 后走 `/embeddings`；留空用本地确定性嵌入。
- **Learned Memory / 汇总 LLM 化**：真实模型可结构化抽取和归纳，mock 时确定性兜底。
- **词级 diff**：已实现 LCS 词级增删高亮。
- **模型可切换**：运营后台模型页可在 Agnes、DeepSeek、Qwen、Moonshot、OpenAI、Claude、mock 间切换。
- **待推进**：时序知识图谱、运营后台项目/报告/知识只读看板、密钥加密/密管、细粒度 RBAC、管理员账号体系和密钥轮换。
