# 产品体系补充审计 · 缺什么 / 多什么 / 加法还是减法

> **续接** `docs/[FABLE5]AUDIT_V6_GLOBAL.md`（问题清单）。本篇回答三个战略问题：
> ① 用户动线是否清晰；② 是否在提供"真正的商业咨询"；③ 能否把用户导向生态周边（AI 知识库经营分析、数字人、AI 短剧等）。
> **结论先行，供后续 agent 按清单执行。** 审计日期 2026-07-04。

---

## 0. 总判断：先做减法，再在三个杠杆点做加法

当前产品体系的病不是"功能少"，是**功能长错了轴**：话术、命理、仪式感、段位这些"叙事层"投入过重，而"数据进 → 诊断 → 处方 → 工具 → 成果回流"这条**钱路上的关键环节是断的**。

- **减法减什么**：并行入口、重复名词、静态目录、过浓的命理叙事、过早曝光的激励体系。
- **加法加什么**：只加三样——**真实经营数据接入、行业基准库、处方引擎（诊断结论→生态工具的结构化桥）**。
- **一句话**：把产品从"内容型（陪聊+算命+仪式感）"换轴到"数据与分发型（数据进、处方出）"。减法是为了让用户看清一条主线；加法是为了让军师的每个建议都能落到一个可开通的工具、并有数据回来证明有效。

## 1. 视角一：用户动线清晰程度

### 1.1 缺什么

| # | 缺口 | 说明 |
|---|---|---|
| L-1 | **全局主线进度（用户级 journey state）** | 用户旅程"建档→诊断第 N 轮→拿方案→执行第 N 天→首次复盘→首月战报"没有任何跨 tab 的统一进度表达。5 个 tab 各自为政，用户不知道"我现在在哪一步、下一步该干嘛"。需要一个用户级状态机（服务端持久化，非会话内轮次计数）+ 所有 tab 顶部同一张「下一步」卡。 |
| L-2 | **10 分钟内的首次价值（速诊产品）** | 现在的入口产品是 6 轮深度诊断——那是"正餐"不是"门"。缺一个 3 问速诊：行业+营收段+最痛的一件事 → 立刻出一张免费「初诊卡」（主要矛盾假设 + 1 条今天就能做的事）。速诊卡是动线的第一格，也是裂变素材（替代合规风险高的"送你一卦"）。 |
| L-3 | **各 tab 之间的因果链可视化** | "对话产生判断→战局展示判断→认可后进执行→复盘刷新战局"这条因果链只存在于设计文档里，界面上没有任何"这条军令来自那份方案的第 3 节"式的血缘展示（军令卡有'来自 xxx 方案'标签，但反向不通：方案页看不到落地率）。用户感知不到闭环，闭环就等于不存在。 |

### 1.2 多余什么（减法清单）

| # | 冗余 | 处置 |
|---|---|---|
| L-4 | **项目（Project）与案卷（Casefile）双容器** | 两个语义高度重叠的"事务容器"并存，用户需要理解"项目里有会话和报告、案卷里有军令和回填"——这是工程演进史泄漏到了 IA。**合并为案卷单主线**（工程上 Project 可保留为底层模型，前台只露案卷一个名词）。 |
| L-5 | **智库 tab 的"市场目录"形态** | 模块市场 + Skill 市场 + 能力市场 + 数据源目录 + 方案库五个并列货架，绝大多数是静态目录（启用=跳对话），逛集市的心智与"军师"人设冲突（参谋不摆摊）。**降级为"已开通能力管理页"**，能力的销售入口全部改为处方式（见视角三）。 |
| L-6 | **三势判断静态框架卡** | 战局页三势卡无结构化数据支撑（无 forces 字段），点击=发起对话，是"看起来有功能"的装饰。要么等结构化落地再上，要么撤下，不要用空框架占用户注意力。 |
| L-7 | **段位/预言在主动线里过早曝光** | 「我的」页战略段位卡对新用户展示"新兵 · 连续复盘 0 天 · 准确率 —%"，是把空账本怼在脸上。冷启动期（<14 天）整卡隐藏，数据攒够再亮相。 |
| L-8 | **名词体系过载** | 报告/方案/成果/卡片/纪要/知识/记忆/档案/军令/模块/Skill——11 个名词描述 4 类东西。用户侧收敛为：**案卷、方案、军令、资料** 四个词，其余全部作为内部术语退出界面文案。 |

## 2. 视角二：真正给用户提供商业咨询

### 2.1 缺什么——咨询的三块地基全缺

| # | 缺口 | 说明 |
|---|---|---|
| C-1 | **真实经营数据接入（最大单点缺口）** | 数据源绑定（淘宝/抖店/小红书/企微）全部是目录引导态，唯一真实通道是财务表上传。没有真实数据，所有诊断的输入都是老板的自述——**自述式咨询=高级陪聊**。不必求全：先打通 1 个高频源（微信支付账单 / 收银 SaaS / 抖店其一），或者退一步先把"手工报数"做成结构化周报表（比现在的三数回填多 5-8 个字段），让军师的判断至少建立在连续数据序列上。 |
| C-2 | **行业基准库（benchmark 数据资产）** | "你的复购率 30%"——然后呢？没有"美业同规模门店中位数是 45%"这句话，前一句毫无咨询价值。**基准数据是咨询与聊天的分界线**。这是内容资产不是功能：admin 已有 12 行业框架可承载，需要运营侧持续维护每行业 5-8 个关键指标的分位数（公开报告+行业协会数据即可起步，不必精确）。prompt 里写死的 6 行业静态认知同时废弃。 |
| C-3 | **知识库 → 经营分析的产品化** | 用户被承诺"AI 知识库帮你做企业和运营分析"，但现状是：上传→解析→切片→被动等对话召回。缺主动分析产品：**上传财务表/报表 → 自动生成「经营体检报告」**（毛利结构、费用异动、现金流预警，deliverable 模板体系现成可挂）；**每月资料自动汇总 → 月度经营简报**。资料库要从"仓库"变成"会出报告的仓库"，否则用户没有第二次上传的理由。 |
| C-4 | **咨询结论的可执行颗粒度** | 军令拆解是分节标题启发式，产出的是"做私域运营"级别的正确废话。真咨询的结论长这样："本周把老客召回短信发给 90 天未到店的 217 人，预期回流 8-12 单"。这要求：十二问结构化落库（字段已预留、管道未建）+ 军令 LLM 结构化拆解（AGENTS.md gap #1）+ 与 C-1 的数据打通。 |
| C-5 | **咨询质量的用户可见证据** | 决策日志/预言账本服务端已建但用户不可见（前审计 F-8）。"我三个月前听军师的砍了那条线，现在数据证明对了"——这是复购和转介绍的唯一素材，现在锁在数据库里。 |

### 2.2 多余什么

| # | 冗余 | 处置 |
|---|---|---|
| C-6 | **命理浓度过高（从主粮降为佐料）** | V6.0 把「势」哲学+八字作为系统脊柱，全流程贯穿。对"真正商业咨询"的定位而言，命理是**获客钩子和仪式感层**，不是分析引擎：留（速诊卡命格一句话、天时日历、报告封面谶语）；降（择时精确到日、称骨、团队八字匹配、流月攻守逐月推演——引擎撑不住且合规风险高）。命理模块全局开关化（前审计 P-3）与本条同一工程动作。 |
| C-7 | **六层复盘体系超配** | 日/周/月/季/年/团队六层，账本只有 day 层、用户 90% 只会用日复盘。**收敛为日打卡 + 周复盘 + 月战报三层**，季/年并入月度线（年度=12 月聚合），团队复盘砍掉（它依赖团队成员数据，单老板产品里是空中楼阁）。 |
| C-8 | **12 专业军师并列目录** | growth/intel/fund/brand…一字排开让用户自己选，等于让病人自己选科室。保留 4-5 个与处方路径绑定的科室（见视角三），其余下架或合并——军师数量不是卖点，被派单到正确军师才是。 |

## 3. 视角三：吸引用户使用周边功能 / 生态引流（数字人、AI 短剧等）

### 3.1 缺什么——从"货架"到"处方"的整条链路

| # | 缺口 | 说明 |
|---|---|---|
| E-1 | **处方引擎（本视角的核心缺环）** | 生态引流的正确形态不是"市场里摆着数字人工具等用户逛"，而是**军师开处方**："你的主要矛盾是获客，你的命格/优势适合做 IP → 破局路径是短视频影响力 → 军令 #2：每周 3 条口播视频 → 你不会拍？用数字人代播，点此开通 💎xN"。这要求方案/军令产出结构化的 `prescription` 字段（问题→打法→工具 key→定价），军令卡上直接挂"一键开通"。**转化发生在诊断的信任峰值上，而不是货架上。** 当前 report→module 关联、军令→工具挂载均无模型（AGENTS.md gap #4 部分覆盖）。 |
| E-2 | **品牌资产包（档案 → 生态产品的输入侧打通）** | 数字人要人设、短剧要品牌调性、短视频要话术——这些全部已经存在于军师档案/战略档案里（行业、定位、客群、老板故事、命格性格）。缺一个「品牌资产包」（Brand Kit）：从档案自动生成 IP 人设卡、话术库、视觉调性，作为数字人/短剧产品的**开箱预填输入**。这是生态产品体验碾压外部竞品的唯一结构性优势——别家数字人冷启动要用户自己想人设，我们的军师已经替他想好了。 |
| E-3 | **成果回流（生态引流的下半场）** | 用户被引到数字人/短剧产品之后，产出数据（视频发布数、播放、留资）必须回流到军师的 KPI/复盘里："上周数字人发了 6 条，带来 23 个线索，占你新增线索的 40%——处方有效，加大剂量。"没有回流，引流是一次性的、军师无法为处方效果背书、用户也不会续费生态产品。需要生态产品间的事件总线/回传接口约定（哪怕 v1 是手工回填+军师追问）。 |
| E-4 | **生态统一身份与账户** | 小程序、数字人平台、短剧工具若各自注册、各自付费，处方链路每跨一步流失一半。需要统一账号（手机号/unionid 打通）+ 钻石/额度跨产品通用（两轴计费体系天然适合作为生态通用货币，这是现有架构里少数为生态化埋好的伏笔——但目前没有任何跨产品结算设计）。 |
| E-5 | **处方转化漏斗埋点** | 处方展示→点击→开通→首次使用→成果回流，这条漏斗是未来生态的核心报表，当前无任何埋点模型。后续运营定价、军师话术优化全靠它。 |

### 3.2 多余什么

| # | 冗余 | 处置 |
|---|---|---|
| E-6 | **静态市场目录（与 L-5 同一动作）** | 模块/Skill 市场的"逛"心智砍掉；已开通能力收进管理页。生态产品**只通过处方和战报出现**——军令里出现、复盘里被点名表扬，就是它的全部曝光位。 |
| E-7 | **与主线无关的创作型智能体 SKU** | 海报、文案等低价值单点工具若不能挂进处方路径（服务于"影响力→引流"这条打法），就是分散心智的僵尸 SKU。每个创作型 agent 必须回答"它出现在哪条处方里"，答不上来的下架。 |

## 4. 执行排序（加减法合并清单）

### 第一批：减法（低成本、先让主线露出来）
1. **L-4** 前台合并项目/案卷为单主线名词（工程模型不动，改 IA 与文案）
2. **L-8** 名词体系收敛为 案卷/方案/军令/资料
3. **L-5+E-6** 智库 tab 改造：市场目录 → 已开通能力管理；撤三势空框架卡（L-6）
4. **L-7** 段位/预言卡冷启动期隐藏
5. **C-7** 复盘收敛三层；**C-8** 军师收敛 4-5 科室
6. **C-6** 命理降浓度 + 全局开关（与合规 P0 同一动作）

### 第二批：加法·咨询地基（决定产品是不是"真咨询"）
7. **L-2** 3 问速诊 + 免费初诊卡（新入口产品，替代"送你一卦"做裂变）
8. **L-1** 用户级 journey 状态机 + 全 tab 统一「下一步」卡
9. **C-2** 行业基准库 v1（12 行业 × 5-8 指标分位数，admin 承载，运营维护）
10. **C-3** 知识库分析产品化：财务表 → 经营体检报告（复用 deliverable 体系）
11. **C-1** 真实数据接入试点 1 个源（或先做结构化周报数表单）
12. **C-5** 决策/预言账本用户可见页（复用前审计 F-8）

### 第三批：加法·生态引擎（决定生态引流成不成立）
13. **E-1** 处方引擎：方案/军令结构化 prescription 字段 + 军令卡挂"一键开通" + **E-5** 漏斗埋点
14. **E-2** 品牌资产包：档案 → IP 人设/话术/调性，作为数字人/短剧的预填输入
15. **E-3** 成果回流 v1：生态产品数据回填 KPI/复盘（可先手工）
16. **E-4** 生态统一账号 + 钻石/额度跨产品结算设计

> **依赖关系**：第三批依赖第二批的 journey 状态机（E-1 的处方要知道用户在哪一步）与档案质量（E-2 的输入）；第二批依赖第一批腾出来的界面与心智空间。**顺序不可倒置：先减、再筑地基、后接生态。**

## 5. 一页话总结

- **动线**：缺一根用户看得见的主线（journey 状态机 + 速诊入口），多的是并行容器和静态货架——**减容器、立主线**。
- **咨询**：缺数据、缺基准、缺可执行颗粒度——没有这三样，咨询是陪聊；命理和六层复盘是超配的叙事装饰——**降叙事、筑地基**。
- **生态**：缺的不是入口而是**处方权**——军师在信任峰值上开单、品牌资产包降低生态产品冷启动、成果回流让处方可验证；多的是让用户自己逛的市场——**撤货架、建处方**。
- **总答案**：**结构性减法 + 三点式加法**。砍掉一半名词和货架，把省下的全部资源押在"数据接入、基准库、处方引擎"三个杠杆点上。
