> 本文件用于交给开发团队 / coding agent，将现有**高保真原型**落地为**真实可投产产品**。

## 0. 一句话产品定义

面向**创始人 / 企业主**的 AI 商业咨询应用。核心主张是「**被输出**」——用户无需准备材料，AI 智能体直接产出可执行的咨询成果（战略诊断、增长方案、融资清单、IP/宣传片/海报等品牌资产）。两层能力：

- **智库（出谋）**：8 位咨询顾问，产出结构化报告与方案。
- **智能体工坊（出活）**：创作类智能体产出品牌资产 + 训练专属智能体。

贯穿三条 AI 交互主线：① 对话即入口 ② 军师主动推送诊断 ③ 多智能体编排 + 持续学习记忆。

## 1. 技术栈建议（可替换，但需满足约束）

| 层 | 建议 | 约束 |
|---|---|---|
| 移动端 | 小程序（Taro / uni-app 跨端）+ H5；或 React Native | 需同时覆盖**微信小程序 + 独立 App**（原型已按此设计） |
| Web 后台 | 运营后台用 React + 移动端布局（原型即移动端形态） | 复用同一套设计令牌 |
| 后端 | Go / Java / Node（按团队），微服务或模块化单体 | 需支撑多租户、计量、审计 |
| AI 网关 | 自建 LLM Gateway（统一封装模型调用、限流、计费、审核） | **必须**，见 §5 |
| 大模型 | 国内可商用模型（通义千问 / 豆包 / DeepSeek / Kimi 等） | 必须为**已备案**的生成式服务，见 §9 |
| 向量库 | pgvector / Milvus / Qdrant | 用于 RAG 与长期记忆 |
| 关系库 | PostgreSQL | 主数据 |
| 对象存储 | S3 兼容（MinIO / 阿里 OSS） | 报告 PDF、生成的图片/视频 |
| 缓存/队列 | Redis + 消息队列（Kafka / RocketMQ） | 异步生成、编排 |

## 4. 前端契约（必须与原型一致）

### 4.1 智能体注册表
原型 `scripts/app.js` 的 `AGENTS` 即智能体清单的事实来源。投产时改为**从后端拉取**（`GET /agents`），字段对齐：`key / name / role / icon / type / greet / chips / mem(记忆条文案) / deliverable_key / enabled`。

当前 13 个智能体：
- 通用：`general`（军师）
- 智库/出谋：`strat 战略诊断官` `growth 增长操盘手` `intel 竞争情报官` `fund 融资参谋` `model 商业模式设计师` `org 组织人效顾问` `brand 品牌营销官` `ops 经营参谋`
- 工坊/出活：`ip 企业IP打造官` `promo 企业宣传片导演` `poster 海报设计师` `shortvideo 短视频策划` `copy 商业文案官`

### 4.2 结构化成果（Deliverable）数据契约
原型 `DELIVERABLES[key]` 定义了产出的**固定结构**，真实模型输出必须约束成此结构（用 function calling / JSON schema）：

```json
{
  "title": "战略诊断报告",
  "icon": "target",
  "meta": "云栖科技 · SaaS / 软件 · A 轮前后",
  "sections": [
    { "h": "现状判断", "b": "一段话…" },
    { "h": "关键卡点", "list": ["…","…","…"] },
    { "h": "30 天行动建议", "list": ["…","…","…"] }
  ],
  "trust": "本成果基于行业基准与你提供的信息生成，供决策参考；重大决策请结合专业意见。",
  "actions": ["save_to_library", "export_pdf"]
}
```
- 每个 `deliverable_key` 有自己的 section 模板与系统提示词（见 §5.2）。
- `{PAIN}` 等变量在运行时由 `profile` 注入（原型已演示个性化插值）。
- **渐进式呈现**：前端按 section 流式渲染（SSE / 分段返回），保留原型的骨架→渐显动效。

### 4.3 会话与隔离规则（已在原型实现，需后端化）
- **每个 (用户, 智能体) 独立会话线程**；点不同顾问进各自会话，互不串味。
- 中间「对话」Tab = **直接开新会话**（默认通用军师）；空会话不落库。
- 「对话」列表可回溯历史；点开还原全部消息继续聊。
- 成果可「存入方案库」，方案库与会话关联（成果 ↔ 产出它的会话可互跳）。

## 5. AI 工程（产品成立的核心）

### 5.1 LLM Gateway（统一网关，必须自建）
所有模型调用经网关：**模型路由、并发限流、Token 计量、输入输出内容审核、结果缓存、故障兜底/降级**。对上层暴露统一接口：`POST /llm/complete`（支持 stream）。

### 5.2 智能体 = System Prompt + 工具 + 记忆 + 产出 schema
每个智能体在后台（`运营后台.html → 顾问详情`）可配置：
- **System 提示词**：定义角色、产出结构、语气；支持变量 `{企业档案} {行业基准} {长期记忆} {本命色}`，运行时注入。原型 `运营后台.html` 的 `AGENTS[i][4]` 已给出每个智能体的初版提示词，**直接作为种子**。
- **产出 schema**：对应 §4.2，用 function calling 强约束，避免自由发挥/胡说。
- **工具**：竞争情报官需联网检索；创作类需调图像/视频生成。

### 5.3 RAG
- 行业基准库、企业自有资料（可选投喂）→ 向量化检索，作为上下文注入。
- 注意来源标注与时效（前端「可信赖页脚」要有真实依据支撑）。

### 5.4 多智能体编排（工作流）
原型「诊断→策略→文案→物料」需真实编排引擎：定义 DAG，节点为智能体调用，支持中间结果传递、失败重试、人工确认。建议用工作流框架或自研状态机 + 队列。

### 5.5 创作类生成
- 海报/封面 → 文生图模型；宣传片/短视频 → 脚本（LLM）+ 可选 AI 视频。成本高、时延长，**必须异步 + 进度反馈**，并做好成本上限。

## 6. Agent Memory（"越用越懂"，你的关键差异点）

- **存储**：`memory` 表（结构化 fact/preference + 向量），按 `(用户, 智能体)` 隔离。
- **写入**：每次会话后由「记忆提炼」任务（LLM 抽取要点）写入；后台「从对话中自动学习」开关控制。
- **检索**：下次产出前召回相关记忆注入 prompt（`{长期记忆}` 变量）。
- **淘汰/留存**：后台「学习强度（保守/均衡/激进）」「记忆留存（30/180天/永久）」映射为真实的权重衰减与 TTL 策略。
- **反馈回流**：成果的采纳/修改/忽略 → 写入 `feedback` 记忆，形成"越用越准"闭环。
- **冲突消解**：新旧记忆冲突时按时间/权重/来源可信度合并。
- 前端「记忆条」「记忆已更新」提示需由真实记忆事件驱动。

## 7. 运营后台（从演示到真控制台）

`运营后台.html` 已定义全部配置界面，投产需让其**真实下发生效**：
- **每日献策**：献策库增删 / 启用 / 排期；每日 08:00 定时任务随机推一条（计入触达）。
- **内置顾问**：启用开关、**System 提示词在线编辑**、**Agent Memory 策略配置**（长期记忆开关、自动学习、学习强度、记忆留存、记忆来源）。改动需版本化、可回滚、灰度下发。
- **建档问卷**：问题与选项可配，前端首登动态渲染。
- **套餐算力**：定价、每月算力、含顾问数、权益；与计费系统打通。
- **概览看板**：付费/活跃/产出/算力消耗等真实指标（接数据仓 / 埋点）。
- 后台自身需 **RBAC**（运营/管理员权限）与操作审计。

## 8. 账号 / 多租户 / 计费

- **登录**：手机号 + 微信授权；企业（租户）与成员关系。
- **多租户隔离**：所有数据 `tenant_id` 行级隔离；企业版支持私有化部署（数据不出内网）。
- **本命色**：首登强制选择（金色默认第一），存用户档案，后续在「我的→本命色」可改；驱动前端主题。
- **计费**：套餐购买（微信支付 / 对公）、**算力按次/Token 实时扣减**（`credit_ledger`）、限流、续费、发票。

## 9. 合规与安全（国内强约束，不可省略）

- [ ] **生成式 AI 服务备案 + 算法备案**（上线前完成，监管硬性要求）。
- [ ] **内容安全审核**：输入与输出双向过滤（敏感词 + 模型审核），留 `moderation_log`。

- [ ] **等保**（按业务等级定级备案）；私有化部署交付方案。
- [ ] **免责与边界**：AI 建议显著标注「供决策参考、重大决策请结合专业意见」（前端已埋，需制度化 + 不得提供持牌业务如证券投顾的违规建议）。

- **可观测**：调用链追踪、LLM 调用日志、成本看板（Token 成本是主开销，需按租户/智能体归集）。

## 11. 里程碑（建议分阶段）

**M0 · 地基（2–3 周）**
账号/租户、登录、会话与方案库云端持久化、LLM Gateway 骨架、运营后台真实读写。

**M1 · 最小可用闭环 MVP（3–4 周）** ⭐ 先验证付费意愿
打通 **1 个智能体（战略诊断官）** 的真实链路：真实模型 + 结构化报告（function calling）+ 个性化（建档注入）+ 方案库 + **真实长期记忆** + PDF 导出 + 计费扣减；完成**备案与内容审核**。可小范围灰度收费。

**M2 · 铺开智库（3–4 周）**
其余 7 位顾问 + RAG 行业基准 + 主动推送（中台/首页洞察）+ 每日献策定时任务。

**M3 · 智能体工坊（4–6 周）**
创作类智能体（IP/宣传片/海报/短视频/文案）接图像/视频生成；专属智能体训练流程。

**M4 · 编排与企业版（按需）**
多智能体工作流编排；企业版私有化部署、私有数据接入、等保。

> 关键路径与依赖：**备案/审核** 是上线前置（与 M1 并行启动，周期可能长，尽早报）；LLM Gateway 是所有 AI 功能的底座（M0 先做）。

## 12. 给 coding agent 的执行说明

1. **先读原型**：`方案A · 本命色.html`、`运营后台.html`、`scripts/app.js`、`styles/*.css`。视觉、交互、文案、数据结构以原型为准。

4. **AI 产出必须结构化**：用 function calling 约束成 §4.2 的 schema，不要让模型自由输出长文。
5. **先 M1 跑通一个智能体的真实闭环**，再横向铺开，避免一次性全接。
6. 所有对外建议带免责边界；所有模型调用经 Gateway 审核与计量。
